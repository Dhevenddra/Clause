# v0.2 design notes (post-competition — DO NOT touch v0.1)

## Reference the founder likes: litmushiring.com (scraped 2026-07-11)
- Type: **Instrument Sans** for headings (h1 64px, h2 48px), Inter body at a large 20px.
  Taste profile: quiet, oversized type; almost no chrome.
- Color: near-monochrome (black on white, grey #9CA3AF accents) — all the color lives in
  **gradient-mesh image panels** (green/red/blue blurs) used as section punctuation, not decoration.
- Components: fully-rounded pill buttons, radius 0 everywhere else, shadow-free, Tailwind.
- Layout: single column, huge negative space, 4px base unit; sections alternate
  text-block / full-bleed gradient panel; booking widget embedded at the end.
- Personality: professional, medium energy. The wow is restraint + the mesh panels.

## What could carry into CLAUSE v0.2 (without losing the audit-ledger identity)
- Bigger, calmer hero type; more negative space between ledger sections.
- One gradient-mesh panel as the hero backdrop behind the stamp (ink-tinted, not rainbow).
- 20px body size for rationale text.
- Keep: verification seals, monospace evidence, stamp palette (they're the product).

## Astryx (github.com/facebook/astryx — Meta design system, React 19 + StyleX)
Founder wants it tried. Requires a React build pipeline → out of scope for the
no-build v0.1 (DEC-005). v0.2 path: Vite + React 19, `@astryxdesign/core` +
`@astryxdesign/theme-neutral` + CLI (`npx astryx init` generates agent docs — read
them first), re-implement the ledger as composed Astryx components with a custom
brand theme (ink/paper/stamp tokens), keep the FastAPI backend untouched.
